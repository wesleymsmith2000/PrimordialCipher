Engine-first architecture

Code Documentation

The MVP is built around a UI-independent TypeScript engine. React renders state and submits actions; game rules live under src/engine/, AI choices under src/ai/, and visual asset mappings under src/skins/.

Module Map

Module Responsibility Reuse Notes
src/engine/types.ts Shared domain types for players, tiles, anchors, actions, state, telemetry, and logs. Start here when porting rules into a simulation, server, or another UI.
src/engine/math.ts Prime-factor math, LCM, products, resonance storage, overlap, and absence counts. Pure functions with no UI dependency. Good candidate for extraction into a shared package.
src/engine/state.ts Initial game state, player records, anchors, starting prime bag, and telemetry. Scenario setup and test fixtures can build on this module.
src/engine/game.ts Action dispatcher and turn-level rule orchestration. UI and AI should call applyAction instead of mutating state directly.
src/engine/network.ts Placement legality, anchor contact, tile connectivity, and recursive network checks. Keep this independent from board rendering so alternate boards can reuse the topology rules.
src/engine/combat.ts Attack eligibility, Harmony, Dissonance, dice scoring, capture, discard, and combat telemetry. Uses injectable dice via DieRoller, which supports deterministic tests and simulations.
src/engine/retromorphosis.ts Retromorphosis eligibility, factor movement, new Unus calculation, discovery, and unlock tokens. Discovery logic is currently in the engine and should stay test-covered as rules mature.
src/engine/primeBag.ts Standard prime contributions, super-prime constants, prime bag creation, and draws. Future seeded randomness can be introduced here without touching React.
src/engine/discard.ts Discard stack capacity and tile archiving. Used by combat, build abandonment, reconstruction, and network loss.
src/engine/tiles.ts Duae and tile constructors plus reveal helpers. Useful for tests, AI-generated moves, and tabletop scenario importers.
src/ai/heuristicAi.ts AI plugin interface and starter heuristic AI. The plugin shape can later support local, remote, or LLM-assisted players.

Current Extension Points

  • AiPlayerPlugin.chooseAction(state, playerId) can drive local AI, scripted tests, or simulations.
  • DieRoller in combat makes dice deterministic for tests and balance sweeps.
  • createInitialState can be complemented by scenario builders for tutorials or regression fixtures.
  • src/skins/ keeps asset registries outside engine rules, supporting alternate visual themes.
  • GameAction is the contract for UI, AI, and future multiplayer commands.

Implementation Guardrails

Keep prime factors, LCM relationships, resonance, combat, retromorphosis, and network topology in engine modules. UI components should render state and submit actions, not decide rules.
  • Add engine tests under src/tests/ when changing behavior.
  • Prefer pure helpers and immutable state updates for rule logic.
  • Use module exports from src/engine/index.ts for public engine APIs.
  • Document intentionally incomplete MVP areas as pending rather than smoothing over them.
